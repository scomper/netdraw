# NetDraw 网络拓扑绘图工具 — 项目总结

## 项目概述
NetDraw 是一款基于纯前端（HTML + CSS + SVG）的交互式网络拓扑绘图工具，用于可视化企业网络设备架构。所有功能在单个 HTML 文件中实现，无外部依赖。

---

## 已完成功能（阶段性总结）

### 一、核心绘图引擎
| 功能 | 状态 | 说明 |
|------|------|------|
| SVG 画布 | ✅ | 网格背景 + 缩放/平移 + 橡皮筋选择 |
| 节点拖拽 | ✅ | 单选/多选拖拽 + 网格吸附（20px） |
| 连线绘制 | ✅ | 锚点拖拽绘制 + 贝塞尔曲线 + 标签 |
| 锚点系统 | ✅ | 四方向锚点（上下左右），选中连线时高亮端点锚点 |
| 连线标签拖拽 | ✅ | 标签可拖拽调整位置，5px 网格吸附 |
| 撤销/重做 | ✅ | Ctrl+Z / Ctrl+Y，最多 50 步（含 groups/templates） |
| 右键菜单 | ✅ | SVG 线条图标（Feather 风格），点击任意处自动关闭 |
| 快捷键 | ✅ | Delete 删除、Ctrl+A 全选、Escape 取消选择 |
| 剪贴板 | ✅ | Ctrl+C 复制、Ctrl+V 粘贴、Ctrl+D 快速复制 |
| 快速保存 | ✅ | Ctrl+S 手动保存（首次弹窗选位置，后续直接覆盖） |
| 框选 | ✅ | 拖拽空白处直接框选（无需 Shift），Shift 改为平移画布 |

### 二、设备类型系统（17 种）
| 分类 | 设备 |
|------|------|
| 云与存储 | NAS、云服务 |
| 网络设备 | 路由器、交换机、光猫、无线 AP |
| 终端设备 | 台式机、笔记本、手机、用户 |
| 安全设备 | 安全设备、防火墙 |
| 服务器与监控 | 服务器、数据库、摄像头、NVR |
| 自定义 | 自定义类型（用户可新建/删除） |

每种类型包含：SVG 图标 + 预设字段 + 独立颜色。

### 三、属性面板
- 节点属性：名称、设备类型、字段编辑（文本/多行/数字/下拉/链接/日期/开关）
- 自定义字段：添加/删除/重命名字段
- IP/MAC 字段：等宽字体 + 深色 + 加粗
- 连线属性：标签、源端点标签（`sourceLabel`）、目标端点标签（`targetLabel`）
- 分组属性：重命名分组

### 四、节点卡片
- 白色圆角卡片 + 图标 + 名称
- 显示多个字段值（最多 2 行，自动适配卡片高度）
- 高层视图下仅显示名称
- 默认 drop-shadow 阴影，hover 时加深

### 五、分组系统
- 框选创建分组 / 右键创建分组
- 分组框自适应包围内部节点（含间距）
- 四角拖拽调整大小（圆形手柄，网格吸附）
- 移动分组时内部节点跟随

### 六、模板系统
- 右键节点 → 保存为模板
- 模板面板：拖拽到画布添加、删除模板（✕ 按钮）

### 七、多画布（多标签）系统
- 工具栏内嵌圆点：每个画布一个圆形编号按钮
- 当前画布紫色高亮，未保存画布红色圆点
- 悬停显示 ✕ 关闭按钮 + 确认对话框（防误删）
- ＋ 新建画布按钮
- 所有画布数据持久化到 localStorage

### 八、多对象操作
- Shift+点击加选/减选，框选多选，Ctrl+A 全选
- 选中对象：设备类型色边框 + drop-shadow 光晕
- 多选对象：设备类型色边框 + 右上角 ✓ 标记徽章
- 多对象面板（2 列网格布局，不折行）：
  - 对齐：左/右/顶/底/水平居中/垂直居中
  - 分布：水平等距/垂直等距 + 固定间距
  - 操作：自动排版、分组、删除

### 九、自动排版
- BFS 分析连线关系确定网络层级
- 根据层级自动排列（保持原有左右顺序）
- 子节点居中于父节点下方
- 重新分配锚点端口

### 十、工具栏设计
- 紧凑左对齐布局（圆角 24px）
- Logo + 文件名（可直接编辑） + 工具按钮
- 画布圆点（内嵌工具栏，与工具栏中心对齐）
- 右上角独立圆形 `?` 帮助按钮（与工具栏中心线对齐）

### 十一、帮助对话框
- 点击 `?` 按钮弹出，右上角对齐
- 内容：基本操作、画布操作、编辑快捷键、批量操作
- 点击遮罩层或 ✕ 关闭

### 十二、AI 拓扑图生成
- 支持模型：通义千问 VL、豆包 VL、OpenAI GPT-4o、SiliconFlow、Ollama 本地
- 支持输入：图片（JPG/PNG/WebP）、JSON 拓扑文件、纯文字描述
- **识别策略**：
  - 图片识别：AI 根据图中设备的相对位置分配 x/y 坐标，保留原图空间关系
  - 纯文字描述：AI 返回层级信息，系统自动按层级布局（无 x/y 坐标时回退）
  - 分组通过设备ID列表引用，坐标由系统根据设备位置自动计算
  - 只连接有实际连线关系的设备，孤立节点保持独立
- 调试工具：`test-ai-topo.py` 可独立测试 AI 识别结果

### 十三、导出/导入
- JSON 导出/导入（文件名跟随标题）
- 保存逻辑：首次弹窗选位置，记住文件句柄，后续直接覆盖
- PNG 截图导出：透明背景 + 内联样式 + 4x 原生光栅化
- SVG 矢量导出：内联样式 + 自动计算 viewBox

### 十四、高层视图
- 完整视图 ↔ 高层级视图切换
- 高层级视图下节点卡片简化

### 十五、性能优化
- localStorage 写入异步化（`setTimeout` 延迟），不阻塞渲染
- 拖拽时增量更新（只改 transform + 相连 edges 路径，不全量重建 SVG）

### 十六、连线语义变体（借鉴 Archify）
- 4 种连线样式：默认（灰色实线）、主干链路（紫色粗线）、安全链路（红色虚线）、异步/备份（灰色虚线）
- 选中连线后属性面板顶部显示彩色圆点按钮，点击切换
- 旧数据自动迁移 `variant: 'default'`
- 导出（PNG/SVG）时根据变体内联对应样式

### 十七、SVG 矢量导出
- 克隆 SVG → 清理交互元素 → 内联所有 CSS 样式 → 自动计算 viewBox → 下载 `.svg` 文件
- 工具栏新增 SVG 下载按钮（下载图标）

### 十八、节点遮罩层（借鉴 Archify c-mask）
- 每个节点卡片内部插入不透明白色遮罩矩形 `.n-mask`，遮挡穿过节点的连线
- 连线渲染在节点之下，但遮罩层确保连线不穿透卡片

### 十九、4× 高清导出
- PNG 截图从 2× 升级为 4× 原生光栅化（浏览器在 4 倍分辨率下渲染矢量再绘制到 canvas）
- 输出更清晰，适合 Retina 屏幕和打印

## 技术亮点
1. **纯前端实现**：单文件 HTML，零依赖，可离线使用
2. **SVG 嵌套图标**：22 种 SVG 图标通过 CSS `color` 属性继承，支持动态着色
3. **导出样式内联**：克隆 SVG 并内联所有 CSS class 样式，确保 data URI 渲染正确
4. **网格吸附系统**：所有操作（拖拽、对齐、分组、自动排版）统一 20px 网格
5. **多画布隔离**：每个画布独立保存状态，切换时完整恢复
6. **剪贴板跨画布**：复制的节点在内存中持久化，切换画布后仍可粘贴
7. **防御性初始化**：localStorage 数据损坏时自动清除并使用默认拓扑
8. **端口标签分离**：`sourcePort`/`targetPort` 存储锚点方向，`sourceLabel`/`targetLabel` 存储用户端口名
9. **智能保存**：记住文件句柄，文件名不变时直接覆盖不弹窗

### 二十、设备面板三态自适应（2026-07-02）
- **ResizeObserver 驱动**：实时监听面板宽度，自动切换 `data-size` 属性，纯 CSS 过渡，不重建 DOM
- **三态布局**：
  - `full`（≥140px）：完整 accordion 分类 + 图标+文字
  - `compact`（60-140px）：双列图标 + 小药丸分类标签（9px 居中圆角浅灰背景）
  - `mini`（<60px）：单列纯图标 + 分隔线分类，标题隐藏
- **拖拽弹性吸附**：右侧把手拖拽，松手吸附到 3 档（96px / 220px），`.dragging` 禁用过渡避免拖拽延迟
- **Tooltip**：compact/mini 模式下悬停显示名称，`position:fixed` 全局定位，不被 `overflow` 裁剪
- **滚动条隐藏**：`.lp-body::-webkit-scrollbar{width:0}`，视觉上不挤压内容区
- **分类分隔线**：`.lp-section` 底部 `border-bottom` 区分分组
- **新建节点按钮**：统一为一个组件 `lp-add-btn`，compact/mini 模式 `gap:0` 保证居中
- **属性面板拖拽**：支持头部拖拽移动，和左侧面板行为一致
- **持久化**：localStorage 保存宽度（net-lp-width）和折叠状态（net-lp-collapsed）

## 设计参考（2026-07-02）
研究了 Figma、Excalidraw、Miro、draw.io、Photoshop 的面板设计：
- 业界普遍采用离散状态切换（非渐进式隐藏）
- Photoshop 的双列/单列模式是最经典的图标模式设计
- 核心结论：离散状态 + CSS 过渡 + 不重建 DOM
